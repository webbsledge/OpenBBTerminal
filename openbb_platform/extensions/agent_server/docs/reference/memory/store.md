# `openbb_agent_server.memory.store`

`MemoryStore` ABC plus the immutable `Memory` record consumed by every retriever, writer, and ingestion helper. All methods are principal-scoped — implementations MUST filter every read and write by `principal.user_id`.

**Source:** [`openbb_agent_server/memory/store.py`](../../../openbb_agent_server/memory/store.py)

## `class Memory(BaseModel)`

One row in the user's cross-thread memory. Returned by every store method and threaded through retrievers and the `MemoryWriter` middleware.

| Field | Type | Default | Purpose |
| --- | --- | --- | --- |
| `memory_id` | `str` | required | URL-safe 12-byte token generated by the store on `write`. Stable lookup key for `pin` / `forget`. |
| `user_id` | `str` | required | Partition key. Set from `principal.user_id` at write time. |
| `text` | `str` | required | Memory body. For ingested chunks this is the chunked text prefixed with `[file:<name> chunk i/N]`. |
| `kind` | `str` | `"fact"` | Routing key. `"fact"` / `"context_text"` route to the text-embedded table; anything ending in `_code` (e.g. `"context_code"`) routes to the code-embedded table when configured. |
| `pinned` | `bool` | `False` | Pinned rows are always returned in `recall()` regardless of vector score, with a boosted score of `1.0`. |
| `source_trace_id` | `str \| None` | `None` | The `trace_id` of the run that wrote this memory — joins back to `traces` for "where did this come from?". |
| `score` | `float \| None` | `None` | Populated only by `recall()`: `[0, 1]` similarity score (cosine for `HashEmbeddings`, reranker `relevance_score` when an NVIDIA reranker is configured). |

## `class MemoryStore(ABC)`

Per-user vector memory ABC. Every method takes a `UserPrincipal`; the implementation enforces `user_id` isolation on both reads and writes.

| Method | Purpose |
| --- | --- |
| `write(principal, text, kind="fact", source_trace_id=None) -> Memory` | Insert one memory. Raises `PermissionError` if the principal lacks the `memory:write` scope. Returns the new `Memory` (without `score`). |
| `recall(principal, query, k=8) -> list[Memory]` | Vector recall, top-`k`. Each returned `Memory` has `score` populated. Pinned rows are merged in unconditionally; reranker-rejected rows fall back to the cosine-only ordering. |
| `list_memories(principal, limit=100) -> list[Memory]` | Newest-first scan of the user's memories. No vector search; used by `GET /v1/memory`. |
| `pin(principal, memory_id, pinned) -> Memory \| None` | Toggle the `pinned` flag. Returns the updated row or `None` if not found / wrong user. |
| `forget(principal, memory_id) -> bool` | Delete one row. Returns `False` if not found or owned by another user. |
| `delete_all_for_user(principal) -> int` | Right-to-erasure — drop every memory for `principal.user_id`. Returns the number deleted. |

## Scopes

- **`memory:write`** — required by `write()`. Issued via `keys issue --scope memory:write` (see [`main.md`](../main.md)).
- **`memory:read`** — required by the `/v1/memory` REST surface (see [`app/router.md`](../app/router.md)).

The `memory:read` scope is enforced at the HTTP layer in [`app/router.py`](../../../openbb_agent_server/app/router.py); the store itself does not gate reads (so `recall_user_memory` can run for any authenticated principal).
